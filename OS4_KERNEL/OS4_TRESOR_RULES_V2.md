# OS4 TRESOR RULEBOOK (V2)

Dieses Dokument definiert die **kompletten Systemgesetze** des OS4 Tresor-/Anatomie-Frameworks.  
Es beschreibt unmissverständlich, wie OS4 funktioniert, denken darf, arbeiten darf, wachsen darf und sich verteidigt.

Jeder Agent, jede Migration, jede API, jede Brand, jede Struktur MUSS sich an diese Regeln halten.

---

# 0. Grundlegende Systemgesetze

## Regel 0 – Der Kernel ist unantastbar
Nichts darf Kernel-Dateien modifizieren, außer:
- Steven (L10)  
- Kernel (L9)  
- Migration Engine (unter Aufsicht)

## Regel 1 – Wissen wird erweitert, nie gelöscht
Alte Dateien werden:
- mit MIG_C markiert  
- archiviert  
- nie entfernt
