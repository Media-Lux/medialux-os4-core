# OS4 Knowledge Lifecycle & Memory – V1

## Zweck
Diese Datei definiert die verbindliche Meta-Governance für:
- Wissensentstehung
- Wissensklassifikation
- Erinnerung
- Priorisierung
- Backup
- Device- & Domain-Kontext

Sie ersetzt keine bestehenden Inhalte, sondern verknüpft sie.

---

## 1. Domain-Awareness (Pflicht)

Jede Wissenseinheit gehört genau einer Domain an:

- OS4-Core
- Core Pulse OS4
- Backoffice
- Mobile (Smartphone / Tablet)

Wissen ohne Domain-Zuordnung ist ungültig.

---

## 2. Knowledge Lifecycle (Pflichtstatus)

RAW  
CAPTURED  
CANDIDATE  
CLASSIFIED  
PRIORITIZED  
IMPLEMENTED  
ARCHIVED  
REJECTED

Migration (MIG_A/B/C) bleibt bestehen und ergänzt den Lifecycle.

---

## 3. Device-Awareness

Wissen entsteht auf:
- Desktop
- Smartphone
- Tablet
- Sprache / Chat

Device ist Kontext, kein Speicherort.

---

## 4. Memory & Retrieval

Archiviertes Wissen bleibt reaktivierbar über:
- Zeit-Trigger
- Domain-Trigger
- Decision-Trigger

Archiv ≠ vergessen.

---

## 5. Decision Memory

Systemrelevante Entscheidungen werden als eigenständige Objekte behandelt:
- Kontext
- Begründung
- Entscheider
- Gültigkeit
- Referenzen

---

## 6. Backup als Lifecycle-Bestandteil

Backup ist Teil des Knowledge Lifecycles und nicht nur Infrastruktur.
Bestehende Backup-Mechanismen werden referenziert.

---

## 7. Geltung

Diese Datei ist bindend für:
- OS4 Tresor
- LiveGPT Instanzen
- Migration
- Intelligence
- Operatoren
