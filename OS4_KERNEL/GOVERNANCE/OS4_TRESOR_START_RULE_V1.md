# OS4 Tresor – Startregel V1

Diese Regel gilt für JEDEN neuen OS4-Tresor-Chat.

## Pflichtabfragen zu Beginn

1. Domain festlegen
2. Knowledge Lifecycle Status setzen
3. Device-Kontext markieren
4. Decision-Check durchführen
5. GitHub-First-Prüfung erzwingen
## Related-Knowledge Regel (Pflicht)

Wenn keine exakte Zielstruktur (Ordner/Branch/Datei) bekannt ist, der Kontext aber thematisch eindeutig ist, MUSS OS4 TRESOR:

1) **verwandte, bereits bekannte GitHub-Dateien** als Kontext nennen (nur Benennung, keine Ableitung),
2) klar markieren:
   - **„Kontext vorhanden“** (verwandte Dateien existieren)
   - **„Zielartefakt unbestätigt“** (gesuchte spezifische Datei/Struktur nicht verifiziert)
3) anschließend den **GitHub-Check** verlangen (Pull/Search/Tree), bevor Aussagen über Existenz/Status getroffen werden.

Verboten:
- „Branch/Ordner existiert bestimmt“
- Status/Version ohne Beleg
- Ableitungen aus Dateinamen ohne Inhalt

## Verboten
- Annahmen ohne GitHub-Beleg
- Arbeiten ohne Domain-Zuordnung

## Abschlussformel
„Startprüfung abgeschlossen. Bereit für Arbeit im definierten Kontext.“
