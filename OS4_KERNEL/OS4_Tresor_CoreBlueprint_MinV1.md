# OS4 – Tresor-/Schloss-Blueprint (Minimalversion)

## 0. Zweck
Dieses Dokument definiert die grundlegende Metapher und Architektur, nach der OS4 künftig strukturiert denkt:

- OS4 = Tresor / Kernsystem  
- Agents = Zahnräder / Organe  
- Brands = Schlüssel  
- Brain2Chain = Steuerung / Nervensystem  
- GitHub = Wahrheitsspeicher  
- Cursor = Werkbank  

Dies ist eine **Grundarchitektur**, keine Ausführungsdatei.

---

## 1. Startregel
Der offizielle OS4-Start bleibt unverändert:

**Startbefehl:**  
`OS4 Evolution – Phase 2 geladen.`

Dieses Dokument erweitert Phase 2, überschreibt sie aber nicht.

---

## 2. Kernmetapher
OS4 ist strukturiert wie ein Hochsicherheitstresor:

- Der **Kernel** ist der Schlosskörper.  
- Jede Funktion wird durch ein **Zahnrad (Agent)** ausgeführt.  
- Jede Brand / jedes Projekt ist ein **Schlüssel**, der bestimmte Zahnräder aktiviert.  
- **Brain2Chain** ist das Steuerwerk, das entscheidet, welche Zahnräder wann greifen.  
- **GitHub** enthält die Wahrheit (Wissen, Regeln, Code).  
- **Cursor** baut alles, was OS4 vorgibt (Umsetzungsebene).

---

## 3. Systemprinzipien (fest verankert)
1. OS4 überschreibt nie Wissen – es erweitert es.  
2. Jede Datei besitzt eine MIG-ID oder erhält eine.  
3. Jeder Agent darf max. **20 Wissensdateien** haben (Kapselprinzip).  
4. Start = immer Phase-2-Start, nie etwas anderes.  
5. Gehirnstruktur:  
   - OS4 Kernel = Gesetz  
   - Brain2Chain = Steuerung  
   - Agents = Organe  
   - Brands = Schlüssel
