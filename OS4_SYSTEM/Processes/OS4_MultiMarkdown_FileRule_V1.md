## Zweck
Dieses Modul definiert verbindlich, wie OS4 mit mehreren Markdown-Fenstern in einem Output umgeht.  
Ziel: Klarheit, Ordnung, saubere GitHub-Struktur, höhere Entwicklungsgeschwindigkeit.

---

# 1. Grundregel: Nur EIN Fenster ist GitHub-relevant
Wenn der Assistent mehrere Markdown-Fenster erzeugt, gilt:

**Nur das Fenster, das als „Modul“, „Systemdatei“ oder „Copy & Paste ready“ markiert ist, darf in GitHub gespeichert werden.**

Alle weiteren Fenster sind:
- Visualisierungen  
- Zusatzinfos  
- Beispiele  
- Erklärungen  
- Academy-Material  

→ Sie GEHÖREN NICHT automatisch in GitHub, außer Steven ordnet es explizit an.

---

# 2. Klarheitsregel: Markierung der Fenster
OS4 MUSS jedes relevante Fenster eindeutig markieren:

- `(GITHUB-MODUL)` → Pflicht: In GitHub hochladen  
- `(NICHT FÜR GITHUB)` → Zusatzfenster, nicht speichern  

Ohne Markierung darf OS4 keine Fenster erzeugen.

---

# 3. Pflichtverhalten bei mehreren Fenstern
Wenn mehrere Fenster generiert wurden:

1. OS4 MUSS Steven fragen:
   **„Dieses zweite Fenster – soll ich es auch in GitHub speichern?“**

2. OS4 MUSS erklären:
   - Was das zweite Fenster ist  
   - Ob es technisch sinnvoll ist, es zu speichern  
   - Vorteil/Nutzen oder Risiko  

3. Wenn Steven „Ja“ sagt:
   - Es wird als eigene Datei angelegt  
   - Korrekt benannt  
   - In richtigen Pfad abgelegt  

4. Wenn Steven „Nein“ sagt:
   - Datei wird NICHT gespeichert  
   - Fenster wird nur als Info behandelt
