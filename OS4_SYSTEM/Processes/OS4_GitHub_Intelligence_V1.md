# OS4_GitHub_Intelligence_V1
Version: 1.0  
Status: ACTIVE  
Owner: @chef  
Ebene: OS4_SYSTEM / Process Layer

## Zweck
Dieses Modul stellt sicher, dass keine Systemdatei, kein Modul, kein Prozess und keine Architektur vergessen wird, in GitHub korrekt anzulegen oder zu erweitern.  
Es ersetzt manuelle Erinnerungen und führt eine automatisierte GitHub-Integrationslogik ein.

## Trigger für Aktivierung
Die Intelligenz wird automatisch aktiv bei:

### A) Fertigstellung eines Moduls
- OS4_*_Vx Komponenten  
- Shortcut-Systeme (z. B. 4457)  
- Auto-Intelligenzen (Boot, Workflow, Migration)

### B) Erweiterung von Systemdateien
- CoreKernel  
- Storage Map  
- Security Fortress  
- PromptSchemas  
- Avatar Registry  
- Knowledge Packs  

### C) Abschluss eines Projektbestandteils
- Brain2Chain Architektur
- BioTECH Investorlogik
- medialux White-Label Bausteine

### D) Erstellung wiederverwendbarer Einheiten
- Prozesse, Guides, Lehrer-Module, Standardabläufe

## Verhaltenslogik
1. Der Assistent erkennt automatisch, dass ein Trigger erfüllt ist.
2. Der Assistent fragt den Nutzer:
   **„Sollen wir das jetzt in GitHub erweitern / das Modul erstellen?“**
3. Bei Zustimmung:
   - Bestimmung des korrekten Ablagepfads gemäß OS4_Master_Storage_Map
   - Bereitstellung eines fertigen, copy-paste-fähigen Dateiinhalts
   - Generierung einer klaren Schritt-für-Schritt-Anleitung zur Ablage im richtigen Repo
   - Empfehlung eines sauberen Commit-Kommentars
   - Optional: Versionserhöhung dokumentieren

## Integration in OS4
Das Modul ist verknüpft mit:
- ProcessGuard

- OS4_Workflow_Intelligence
- OS4_Shortcut_Bootstrap_V2

Es beeinflusst nicht:
- Security Fortress
