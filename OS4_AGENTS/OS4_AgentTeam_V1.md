# OS4 – AgentTeam Blueprint (V1)

Dieses Dokument definiert alle internen Agenten (Organe) von OS4,  
ihre Rollen, Ebenen (Level 1–10), Wissenskapselung (max. 20 Dateien)  
und ihre Verbindungen zu GitHub, Cursor und APIs.

Es ist die **Organübersicht des OS4-Körpers**.

---

# 0. Ziele & Prinzipien

- Jeder Agent erfüllt **genau eine Funktion**.  
- Jeder Agent besitzt **maximal 20 Wissensdateien** (Kapselprinzip).  
- Jeder Agent ist **API-fähig** und updatebar.  
- Jeder Agent ist **duplizierbar** für Brands / White-Label.  
- Jeder Agent ist Teil der Tresor-/Anatomie-Struktur.  
- Agents kommunizieren nie direkt untereinander, nur über:
  - **Brain2Chain (Steuerwerk)**
  - **OS4 Kernel-Regeln**
  - **GitHub-Wissensdateien**
